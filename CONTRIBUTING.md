# Contributing to Home Server Pro

First off, thank you for considering contributing to Home Server Pro! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

Please take a moment to review this document before submitting your contributions.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please report any inappropriate behavior to the project maintainer.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please check the existing GitHub Issues to see if it has already been reported. If it hasn't, open a new issue and include:
* A clear and descriptive title.
* Steps to reproduce the problem.
* Specific details about your environment (OS, hardware, software version).
* Expected vs. actual behavior.

### Suggesting Enhancements
We welcome ideas for new features! To suggest an enhancement, please open an issue and:
* Explain the problem you are trying to solve or the gap you want to fill.
* Describe how the feature should work.
* Provide examples or use cases where this feature would be helpful.

### Submitting Pull Requests (PRs)
Ready to contribute code? Follow these steps to ensure a smooth review process:

1. **Fork the Repository:** Create your own fork of the project.
2. **Create a Branch:** Create a feature branch with a descriptive name (e.g., `feature/add-dark-mode` or `fix/connection-bug`).
3. **Keep it Attribution-Compliant:** Remember that this project is licensed under the MIT License. Ensure your code does not violate any third-party copyrights.
4. **Write Clean Code:** Follow the existing code style and formatting patterns used throughout the repository.
5. **Commit Messages:** Write clear, concise commit messages that explain *what* changed and *why*.
6. **Open a PR:** Submit your pull request against the `main` branch. Provide a clear description of the changes you made and reference any related issues (e.g., `Closes #12`).

## Style Guide & Standards

* Ensure your code is thoroughly tested in your local home server environment before submitting.
* Keep documentation and code comments clear, especially for backend infrastructure changes.

Thank you again for your time and effort!
